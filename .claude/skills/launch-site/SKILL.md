# Launch Site — Auto-SEO 一键建站

## Purpose

输入一个热点话题名和域名，自动完成从关键词研究到站点上线的全流程。
目标：1 小时内完成一个 25+ 页面的 SEO 站点部署。

---

## Usage

```
/launch-site TopicName --domain=trytopic.com --description="Brief topic description"
```

### Required arguments

- `TopicName` — 热点话题名称（如 HappyHorse, DeepSeek, Sora 等）
- `--domain=` — 目标域名（如 tryhappyhorse.xyz）
- `--description=` — 一句话描述（用于关键词研究和内容生成）

### Optional arguments

- `--handoff-url=` — 引流目标 URL（默认 https://www.elser.ai/zh/ai-image-animator）
- `--skip-domain` — 跳过域名购买和 DNS 配置（域名已有时使用）
- `--skip-images` — 跳过图片生成（后续补充时使用）
- `--resume-from=<phase>` — 从指定阶段恢复执行（phase1-phase8）
- `--dry-run` — 只输出计划，不执行任何操作

---

## Execution Phases

### Phase 1: Topic Config 生成（< 1 min）

1. 在 `topic-configs/` 目录下创建 `{topic}.json` 配置文件
2. 配置包含：

```json
{
  "topicName": "HappyHorse",
  "domain": "tryhappyhorse.xyz",
  "siteName": "HappyHorse AI Video Guide",
  "gtmId": "GTM-XXXXXXX",
  "baiduVerification": "",
  "indexNowKey": "<auto-generated-32-char-hex>",
  "leadMagnetFormAction": "https://script.google.com/macros/s/XXX/exec",
  "handoff": {
    "baseUrl": "https://www.elser.ai/zh",
    "primaryPath": "/ai-image-animator",
    "utmSource": "tryhappyhorse.xyz"
  }
}
```

3. 设置环境变量 `TOPIC={topic}` 供后续阶段使用
4. 生成 IndexNow key 文件到 `public/{key}.txt`

### Phase 2: Keyword Matrix（< 5 min）

1. 调用 `/keyword-research` skill 或 `scripts/generate-keyword-matrix.mjs`
2. 输入：topic name + description + niche
3. 输出 `keyword-matrix.json`，包含 6 个类别：

| 类别 | 英文 key | 示例数量 |
|------|----------|---------|
| 品牌认知 | brandAwareness | 5-8 |
| 功能教程 | features | 5-7 |
| 对比评测 | comparisons | 5-8 |
| 应用场景 | useCases | 5-8 |
| 技术深度 | technical | 2-3 |
| 争议热点 | controversy | 2-3 |

4. 每个关键词条目格式：

```json
{
  "keyword": "HappyHorse vs Sora",
  "intent": "comparison",
  "suggestedSlug": "happyhorse-vs-sora",
  "suggestedPageType": "comparison",
  "priority": "P0",
  "competitorName": "Sora"
}
```

### Phase 3: Content Generation（< 15 min）

1. 读取 `keyword-matrix.json`
2. 按优先级排序：P0 先生成，P2 最后
3. 为每个关键词生成 MDX 文件到 `src/content/pages/`

**MDX 文件必须包含的 frontmatter 字段（Zod schema 强制）：**

```yaml
title: string
description: string
h1: string
topic: string
page_type: [homepage|explainer|comparison|tutorial|use_case|prompt_collection|tech_analysis|brand_info|...]
intent_cluster: string
keyword_primary: string
keyword_secondary: [string, ...]
model_name: string
competitor_name: string  # 对比页必填
cta_type: [try_now|compare_now|get_prompts|learn_more|get_tutorial|...]
conversion_target: [generation_activation|alternative_evaluation|creator_activation|...]
generation_entry_mode: [direct|indirect|none]
trend_score: 0-100
publish_priority: [Must|Should|Backlog]
route:
  path: /{slug}/
  slug: {slug}
  is_homepage: false
  is_canonical_app_entry: false
canonical_path: /{slug}/
answer_summary: string
source_urls: [url, ...]
fact_status: [verified|mixed|unknown]
fact_status_note: string
last_updated_at: YYYY-MM-DD
disclaimer_text: string
images:
  hero_path: https://assets.dev.tool.elser.ai/generated/{slug}.png
  hero_fit: cover
  hero_frame: balanced
  alt: string
related_pages: [path, ...]  # >= 3
key_facts: [{label, value, status}, ...]  # >= 3
faq_items: [{question, answer}, ...]  # >= 3
```

**必须生成的核心页面（P0）：**
- `home.mdx` — 首页（is_homepage: true）
- `what-is-{topic}.mdx` — 科普解释页
- `try-{topic}.mdx` — 入口页（is_canonical_app_entry: true）
- `{topic}-vs-{competitor1}.mdx` — 头部对比页（至少 3 个竞品）
- `{topic}-how-to-use.mdx` — 基础教程
- `{topic}-prompts.mdx` — Prompt 模板页
- `best-ai-{category}-models.mdx` — 品类榜单页

4. 生成后运行 `npm run build` 验证所有页面通过 schema 校验
5. 自动建立 `related_pages` 交叉链接图

### Phase 4: Image Generation（< 15 min）

1. 运行 `npm run generate:assets`（或 `scripts/generate-assets.mjs`）
2. 为每个没有 hero image 的页面生成图片
3. 使用 Task Service API + `gemini-3.1-flash-image-preview` 模型
4. 每张图片 prompt 规则：
   - 匹配页面意图
   - 电影/编辑风格
   - 暗色高级背景
   - 无文字、无水印
   - 16:9 比例
5. 生成后上传到 S3 CDN（`scripts/upload-to-s3.mjs`）
6. 更新 MDX frontmatter 中的 image path 为 CDN URL

**关键环境变量（从 `../elser-quene/elser-tool-api/.env.local` 读取）：**

```
TASK_SERVICE_BASE_URL
TASK_SERVICE_API_KEY
AWS_REGION
S3_BUCKET
CDN_DOMAIN
AWS_ACCESS_KEY_ID（运行时传入）
AWS_SECRET_ACCESS_KEY（运行时传入）
AWS_SESSION_TOKEN（运行时传入）
```

### Phase 5: Domain Setup（< 10 min）

1. 检查域名可用性（GoDaddy API）
2. 如果可用且 `--skip-domain` 未设置：
   - 购买域名（需要 `GODADDY_API_KEY`, `GODADDY_API_SECRET`）
   - 配置 DNS：A 记录 `@` → `76.76.21.21`, CNAME `www` → `cname.vercel-dns.com`
3. 如果域名已拥有：只配置 DNS

### Phase 6: Build & Deploy（< 5 min）

1. `TOPIC={topic} npm run build` — 构建静态站点
2. `npx vercel link --yes --project={topic} --scope=elser-team` — 链接 Vercel 项目
3. `npx vercel --prod --yes` — 部署到生产环境
4. `npx vercel domains add {domain}` — 绑定自定义域名
5. 等待部署完成，验证 `https://{domain}/` 返回 200

### Phase 7: Search Engine Submission（< 5 min）

1. **IndexNow** — 批量推送所有 URL 到 Bing/Yandex/Naver/Seznam
2. **百度** — 通过推送 API 提交所有 URL（如果有 BAIDU_PUSH_TOKEN）
3. **Google** — 提交 sitemap URL 到 GSC API（如果有凭证）
4. 执行：`node scripts/submit-to-search-engines.mjs`

### Phase 8: Verification（< 5 min）

1. 检查所有页面 HTTP 200
2. 检查 sitemap.xml 可抓取
3. 检查 robots.txt 正确
4. 检查 JSON-LD、OG tags、canonical 完整
5. 检查所有 hero 图片 CDN URL 可访问
6. 输出最终报告

## Handoff URL Configuration

所有 Elser.ai 引流链接使用 UTM 参数追踪：

```
https://www.elser.ai/zh/{path}?utm_source={domain}&utm_medium=referral&utm_campaign={campaign}
```

Campaign 按页面类型区分：
- `homepage_hero` — 首页 Hero CTA
- `image_to_video` — 图生视频工具
- `anime_art` — 动漫生成
- `character_maker` — 角色设计
- `storyboard` — 分镜工具
- `story_studio` — Story Studio

---

## Error Recovery

如果某个阶段失败：

```bash
# 从第 4 阶段（图片生成）恢复
/launch-site TopicName --domain=trytopic.com --resume-from=phase4

# 只重跑搜索引擎提交
/launch-site TopicName --domain=trytopic.com --resume-from=phase7

# 只重跑健康检查
/launch-site TopicName --domain=trytopic.com --resume-from=phase8
```

---

## Caveats

1. GoDaddy API 需要生产环境 key（开发环境 key 无法实际购买域名）
2. 图片生成是串行的（~30s/张），25 张约需 12 分钟
3. DNS 传播可能需要额外 5-10 分钟
4. 百度推送 API 有每日配额限制
5. Vercel 免费版最多 3 个项目，Pro 版无限制
6. Google Search Console API 需要 OAuth2 service account
