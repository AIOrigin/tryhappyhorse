# /keyword-research 命令

## 使用方式

```bash
# 基础用法：输入热点话题
/keyword-research Happy Horse

# 指定语言（默认中文）
/keyword-research Happy Horse --lang=en
/keyword-research Happy Horse --lang=zh  # 默认

# 指定深度
/keyword-research Happy Horse --depth=quick     # 仅关键词矩阵
/keyword-research Happy Horse --depth=standard  # 关键词 + 页面模板（默认）
/keyword-research Happy Horse --depth=full      # 关键词 + 页面模板 + 内容大纲

# 指定行业/赛道（辅助精准定位）
/keyword-research Happy Horse --niche="AI视频生成"

# 组合使用
/keyword-research "某某产品" --lang=zh --depth=full --niche="跨境电商"
```

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `topic` (必填) | - | 热点话题、品牌名、产品名 |
| `--lang` | zh | 输出语言：zh（中文）/ en（英文）/ both（双语） |
| `--depth` | standard | quick = 仅关键词；standard = +页面模板；full = +内容大纲 |
| `--niche` | 自动推断 | 行业/赛道，帮助精准匹配关键词意图 |

## 核心方法论

本 skill 遵循以下关键词研究框架：

### 思维模型：从热点到长尾的四层漏斗

```
热点事件/话题
  ↓ Layer 1: 背景调研（是什么、谁做的、为什么火）
  ↓ Layer 2: 搜索意图拆解（6 类意图矩阵）
  ↓ Layer 3: 模板化关键词模式（变量 × 模式 = 批量词）
  ↓ Layer 4: pSEO 页面模板设计（URL + 结构 + 变量）
```

### 六类搜索意图矩阵

每个热点话题必须覆盖以下 6 类意图：

| 意图类型 | 用户心理 | 关键词模式 | 流量价值 |
|----------|----------|------------|----------|
| **品牌认知 (What)** | "这是什么？谁做的？" | {话题} + 是什么/介绍/背后团队 | 引流顶部 |
| **功能教学 (How)** | "怎么用？怎么上手？" | {话题} + 怎么用/教程/入门指南 | 中部流量 |
| **对比决策 (Which)** | "和 XX 比哪个好？" | {话题} vs {竞品} / 排行榜 | 高转化 |
| **场景应用 (Use Case)** | "能用来做什么？" | 用 {话题} 做 {场景} | 长尾精准 |
| **技术深度 (Deep Dive)** | "原理是什么？怎么实现的？" | {话题} + 架构/原理/源码 | 开发者圈层 |
| **争议热点 (Controversy)** | "靠谱吗？是炒作吗？" | {话题} + 真实评价/是不是骗局 | 短期爆发 |

## 执行流程

### Step 1: 热点背景调研

**目标**: 搞清楚话题的本质，为关键词定位提供上下文。

**操作**:
1. 使用 WebSearch 进行 **3 轮搜索**：
   - 第 1 轮: `"{topic}" 2026` — 获取最新动态
   - 第 2 轮: `"{topic}" 是什么 介绍` — 了解基本定义
   - 第 3 轮: `"{topic}" 评价 争议` 或 `"{topic}" review` — 了解舆论方向
2. 使用 WebFetch 抓取 **2-3 个高质量信息源**（优先官网、知乎、36kr、少数派等）
3. 从调研中提取:
   - **一句话定义**: {topic} 是什么
   - **爆火原因**: 3-5 个核心引爆点
   - **所属赛道**: 行业/领域归类
   - **关键实体**: 相关品牌、人物、竞品、技术名词
   - **争议点**: 质疑和讨论焦点
   - **时间线**: 关键事件节点

**输出格式**:
```markdown
## 话题背景摘要

**话题**: {topic}
**一句话定义**: {definition}
**所属赛道**: {niche}
**爆火时间**: {date}
**爆火原因**:
1. {reason_1}
2. {reason_2}
3. {reason_3}

**关键实体**:
- 团队/公司: {entities}
- 竞品: {competitors}
- 关键人物: {people}
- 技术名词: {tech_terms}

**争议焦点**: {controversies}
```

### Step 2: 关键词矩阵生成

**目标**: 基于 6 类意图 × 背景实体，系统性生成长尾关键词。

**操作**:
1. 对每类意图，结合 Step 1 提取的实体，按以下模式生成关键词：

**品牌认知类**:
```
{topic} + 是什么
{topic} + 哪家公司的
{topic} + 创始人/团队
{topic} + 和 {关联公司} 什么关系
{topic} + 为什么火/爆火原因
{topic} + {中文名/英文名/别名}
```

**功能教学类**:
```
{topic} + 怎么用/使用教程
{topic} + {核心功能} + 教程
{topic} + 新手入门指南
{topic} + 注册/下载/安装
{topic} + {功能A} 怎么操作
{topic} + prompt/提示词模板
{topic} + api 接入/本地部署
```

**对比决策类**:
```
{topic} vs {竞品A}
{topic} vs {竞品B} 哪个好
{topic} 和 {竞品} 对比评测
{年份} + {赛道} + 排行榜/推荐
最好的 {赛道} 工具 {年份}
{topic} 替代品/平替
```

**场景应用类**:
```
用 {topic} 做 {场景A}
{topic} + {行业} + 应用
{topic} + {人群} + 怎么用
{topic} 适合 {用途} 吗
{topic} 做 {场景} 效果怎么样
```

**技术深度类**:
```
{topic} + 技术架构/原理解析
{topic} + 模型/算法/参数
{topic} + 开源/github/源码
{topic} + 本地部署/硬件要求
{topic} + 二次开发/微调
```

**争议热点类**:
```
{topic} + 真实评价/效果怎么样
{topic} + 是不是炒作/营销
{topic} + 靠谱吗/值得用吗
{topic} + 缺点/不足
{topic} + 会取代 {现有方案} 吗
```

2. 使用 WebSearch 验证关键词的真实搜索热度：
   - 对每类意图取 2-3 个代表性关键词，搜索观察结果数量和质量
   - 如果搜索结果极少（< 3 条相关），降低该词优先级
   - 如果发现大量高质内容已占位，标注竞争度高

**输出格式**: 按意图分类的关键词表格（参见报告模板）

### Step 3: 关键词优先级排序

**目标**: 为每个关键词评估价值，给出执行优先级。

**评估维度**:

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 搜索需求 | 30% | 搜索结果数、自动补全是否出现 |
| 竞争难度 | 25% | 首页结果质量、是否有权威站点占位 |
| 商业价值 | 25% | 是否靠近转化（对比 > 教程 > 科普） |
| 时效窗口 | 20% | 话题热度衰减速度、是否需要抢时间 |

**优先级定义**:
- **P0 (立即做)**: 高需求 + 低竞争 + 高商业价值，或话题热度窗口即将关闭
- **P1 (本周做)**: 中高需求 + 中竞争，有稳定长期流量
- **P2 (有余力做)**: 细分长尾、技术深度内容，流量较小但精准
- **P3 (观望)**: 需求不确定或竞争过高，暂缓

### Step 4: pSEO 页面模板设计（depth=standard 及以上）

**目标**: 为高优先级关键词设计可批量生成的页面模板。

**操作**:
1. 将关键词按模式聚类，识别可模板化的 URL 结构
2. 为每个模板设计：
   - URL 模式: `/{prefix}/{variable-slug}`
   - SEO 标题模板: 含核心关键词 + 年份 + 吸引点击的修饰词
   - 页面结构: H2/H3 大纲
   - 变量列表: 该模板需要的所有变量值
   - 数据来源: 变量数据从哪里获取

**典型模板类型**:

| 模板类型 | URL 模式 | 适用意图 |
|----------|----------|----------|
| 对比页 | /compare/{topicA}-vs-{topicB} | 对比决策 |
| 教程页 | /tutorial/{topic}-{feature} | 功能教学 |
| 场景页 | /use-case/{topic}-for-{scenario} | 场景应用 |
| Prompt/模板库 | /prompts/{topic}-{style} | 工具型内容 |
| 百科页 | /wiki/{topic}-{subtopic} | 品牌认知 |
| 排行榜页 | /best/{category}-{year} | 对比决策 |

### Step 5: 内容大纲生成（depth=full 时）

**目标**: 为每个页面模板产出可直接交给写手/AI 的内容大纲。

**操作**:
对每个模板的前 3-5 个高优先级页面，生成：
- 完整 H1-H3 大纲
- 每个段落的核心要点（2-3 bullet）
- 内链建议（链接到哪些其他模板页面）
- CTA 建议（引导用户做什么）
- 字数建议

## 输出文件


## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 话题太新，搜索结果极少 | 基于有限信息推断，标注"数据不足，建议 3-7 天后复查" |
| 话题太泛（如"AI"） | 提示用户加 --niche 参数缩小范围 |
| 英文话题用中文搜 | 自动补充英文搜索轮次 |
| 搜索 API 限制 | 减少验证轮次，标注"部分关键词未验证" |
