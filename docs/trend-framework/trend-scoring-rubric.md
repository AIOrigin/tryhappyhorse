# Trend Scoring Rubric

Use this artifact before creating any new trend page set.

## Required input record

Capture these inputs first:

1. trend name
2. first seen date
3. source URLs
4. likely search intent
5. commercial relevance
6. template fit

## Weighted scorecard

Score every trend on a 100-point scale using the exact framework weights.

| Dimension | Weight | What to judge |
| --- | ---: | --- |
| Audience relevance | 30 | Does this trend match the audience the site already serves? |
| Search intent strength | 25 | Are explainer, comparison, access, prompt, or alternatives queries already visible? |
| Templating ability | 20 | Can the existing page model and CTA taxonomy cover this topic without new architecture? |
| Speed window | 15 | Is this early enough that a fast page set still has ranking and traffic upside? |
| Monetization fit | 10 | Can the traffic move into the site's existing CTA and conversion paths? |

## Decision thresholds

Apply the same publish rule every time:

1. `80+`: publish within 24 hours
2. `60-79`: add to this week's queue
3. `<60`: monitor only

## Reuse guardrails

Only move forward when the topic can reuse the current system as-is:

1. one page per primary intent
2. one primary CTA per page
3. visible non-official disclaimer where brand risk exists
4. facts can be marked as verified, mixed, or unknown
5. no new scheduler, dashboard, or platform feature is required
